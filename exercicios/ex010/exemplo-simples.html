Joga a âncora, marujo!

Podemos e devemos utilizar links para direcionar o usuário a diferentes páginas do nosso site. Para que isso ocorra, utilizamos a tag de âncora <a> em conjunto com o seu principal atributo: href. Combinando os dois montamos a seguinte estrutura básica:

<a href="link">texto<a>

Podemos construir o seguinte exemplo que leva ao repositório git do Professor Gustavo Guanabara:

<a href="https://gustavoguanabara.github.io">Acesse o perfil do Professor Gustavo<a>

Cuja representação em HTML está logo abaixo:

Acesse o perfil git do Professor Gustavo

Do you speak english? (Você fala inglês?)

Podemos também utilizar outro atributo de tag de âncora, o hreflang, que permite indicar qual é o idioma principal do site para onde o link está desviando o fluxo de navegação. Para este caso, temos o seguinte exemplo que leva ao site da W3 Schools:

<a href="https://www.w3schools.com/html/" hreflang="en">site da W3Schools (em inglês)<a>

Site da W3Schools (em inglês)

Mira no Alvo!

Outro atributo que podemos utilizar, é o target, que suporta os seguintes valores:

_blank:
Abre o link em uma nova janela em branco.
_self:
Abre o link em na janela ou frame atual(padrão).
_top:
Vai desfazer todos os frames e abrir o destino no navegador completo.
_parent:
Similar ao uso do _top em referência à janela mãe
framename:
Caso esteja usando frames, indica o nome da janela a abrir.

Vamos nos baser somente ao uso das duas primeiras opções _blank e _self, visto que o uso de frames está em desuso. Assim, temos os seguintes exemplos:

<a href="index.html" target="_self" hreflang="pt-br">Ir para a página inicial<a>

Ir para a página inicial

<a href="https://gustavoguanabara.github.io" target="_blank" hreflang="pt-br">Ir para a página Git do Professor Gustavo (abre em uma nova guia)<a>

Ir para a página Git do Professor Gustavo (abre em uma nova guia)

Esse link é seu ou dos outros?

Existe um recurso bem interessante para links que é indicar qual é a natureza do destino usando o atributo rel. Esse atributo aceita vários valores, entre eles vou citar:

next:
indica que o link é para a próxima parte do documento atual.
prev:
indica que o link é para a parte anterior do documento atual.
author:
indica que é um link para o site do autor do artigo atual.
external:
indica que é um link para outro site que não faz parte do site.
nofollow:
indica que é um link para um site não endossado, como um link pago.

Exemplos:

<a href="pagina2.html" target="_self" rel="next">Continuar navegando no site<a>

Representação em no navegador

Continuar navegando no site.

Neste primeiro exemplo, chamamos o link de link local ou link interno, já que ele leva o visitante a outra página dentro do nosso próprio site. Note que não é necessário nem indicar a URL completa nesses casos.

<a href="https://gustavoguanabara.github.io" target="_blank" rel="external">Ir para a página Git do Professor Gustavo (abre em uma nova guia)<a>

Representação no navegador

Ir para a página Git do Professor Gustavo (abre em uma nova guia)

Neste segundo exemplos chamamos o link de link externo. Nestes casos, devemos indicar a URL completa, incluindo o protocolo http:// ou https:// e o caminho que leve à uma página específica, se for necessário.

e para fazer downloads?

Basta fazer o link diretamente para o arquivo que se deseja efetuar o download e adicionar o atributo download com o valor configurado para o nome do arquivo a ser baixado e o atributo type para indicar ao navegador que tipo de arquivo está sendo baixado. Vamos ver um exemplo:

<a href="arquivos/meulivro.pdf" download="arquivo.pdf" type="application/pdf">Baixe aqui o pdf do livro do professor.<a>

Baixe aqui o PDF do livro do Professor
